# ONCUT 로드맵 — 다음 플랜

> 기록일 2026-06-01 · 현재 v0.12.0 · 단일 진실은 [CLAUDE.md](./CLAUDE.md)

## 0. 비전 (제품 목표, 흔들지 말 것)
**대화형 AI가 적극 개입하는 영상 편집기.**
- **초보자**: 영상 하나 던지면 AI가 컷·자막·정리·쇼츠변환까지 → 고품질 결과.
- **전문가**: AI가 러프컷·자막을 30초에 만들어 작업시간을 비약적으로 단축 →
  **캡컷으로 핸드오프**해 마무리.
- 노선 = "캡컷 기능을 다 따라잡기"가 **아님**. "AI 자동화 + 극단적 편의 +
  캡컷 연동". 부족한 전문 작업은 캡컷에 넘긴다.

## 1. 현재 상태 (v0.12.0까지 완료)
- 엔진: 클립 타임라인(재정렬·분할·트림), 트랜지션(xfade 2패스), **멀티소스
  짜깁기(영상+영상/이미지)**, 자막(스타일·의존명사 청크·remap), 로고/이미지
  오버레이(페이드·구간), 효과음(SFX), BGM(amix·페이드), 텍스트박스(애니메이션),
  형식(쇼츠 9:16/정사각, cover-crop), 루드니스 -14LUFS, 480p 프록시 프리뷰.
- AI: 무음(의도적 쉼 구분)·잔말·반복 자동 컷, ASR 자막 자동.
- 대화형: 자연어→편집 액션. **Claude CLI 우선 → DeepSeek 폴백**(`app/llm.py`,
  `/api/assist`, `/api/admin` 키 입력).
- UX: 풀스크린 앱 셸(상단바/프리뷰/인스펙터/타임라인, 페이지 스크롤 0), AI 시작
  화면(인테이크), 30초 사용법 가이드.
- 모드 B(이미지→내레이션 TTS)도 같은 편집기로 수렴.
- 검증: tests/smoke.py(16+ 체크), CDP 헤드리스 회귀.

## 2. 비전 대비 객관적 진단 (≈40~60% MVP)
| 축 | 진척 | 한 줄 |
|---|---|---|
| 초보자 고품질 | ~65% | 이미 실용적, 출력 품질 OK |
| **AI 적극 개입** | ~35% | 아직 "명령 수행기" — 내용을 못 봄 |
| **전문가 시간단축 + 캡컷 연동** | ~30% | 핸드오프가 컷+자막만, 다리가 얇음 |
| 실시간 미리보기 | 약함 | 근사치(소스 스크럽), 진짜 결과는 렌더해야 봄 |

## 3. 다음 플랜 (우선순위 — 비전 레버리지 순)

### ★ P1. AI를 "내용 인지" 편집 파트너로 (최우선·핵심)
- 지금 LLM은 길이·형식만 받음 → **대본(ASR 세그먼트)·컷 분석·씬 목록을
  컨텍스트로 전달.**
- 제안형/실행형 액션 확장: "지루한 앞부분 잘라", "핵심 3구간 하이라이트",
  "강조 문장마다 자막 키워", "30초 숏폼으로 줄여", "이 영상 요약 편집".
- 새 액션 타입: trim_clip/keep_ranges, highlight, caption_emphasis, summarize.
- `app/llm.py` SYSTEM 프롬프트에 컨텍스트(transcript) + 액션 스키마 확장.
  `/api/assist` 가 job의 cues/segments/cuts를 ctx로 주입.
- 효과: 비전의 "AI 적극 개입"이 실제로 구현됨. 기술 난도 낮음(데이터만 먹이면 됨).

### ★ P2. 캡컷 핸드오프 충실화 (전문가 가치)
- 현재 draft.py = 컷 순서 + 자막(.srt)만. → **텍스트박스·오버레이(이미지)·
  배경음악·트랜지션을 가능한 만큼 캡컷 draft_content.json에 매핑.**
- pycapcut 한계 조사: 텍스트/이미지/오디오 트랙·키프레임 지원 범위 확인 후 매핑.
- 매핑 불가 항목은 "캡컷에서 추가" 안내(과약속 금지). 영상 경로 동기화 가이드(§7).
- 효과: "AI 러프컷 → 캡컷 마무리" 가 진짜 시간단축이 됨.

### ★ P3. 실시간 합성 프리뷰 (신뢰·체감)
- 현재: 편집기 재생은 근사치, 진짜 결과는 480p 프록시 렌더.
- 목표: 캔버스/WebCodecs로 자막·텍스트·오버레이·트랜지션을 **브라우저에서 실시간
  합성** 재생(오디오 페이드 제외 가능). 멀티소스 전환도 매끈하게.
- 난도 높음(있으면 "파는 힘"이 크게 상승). P1·P2 후.

### P4 이후 (선택·심화)
- LLM 엔진: DeepSeek 키 실연결 검증, 응답 캐시·스트리밍, 토큰 만료 자동 폴백 점검.
- 멀티트랙(영상 위 영상 PIP), 키프레임(위치·크기·투명도), 속도조절.
- 컬러 프리셋/필터, 음성 다양성(유료 TTS: OpenAI/Google/ElevenLabs — 엔진 추상화됨).
- GPU 렌더(NVENC/VAAPI)로 추출 속도.
- 동영상 B-roll PIP, 컬러 이모지 번인(Noto Color Emoji).
- 안정성: 상태 영속 강화, 에러 복구, 대용량/동시 처리.
